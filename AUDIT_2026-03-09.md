# hn-daily-skill 深度分析报告（2026-03-09）

## 1. 总体结论

当前仓库可用，但存在“生成链路不一致 + 内容完整性依赖人工”的核心风险。

- ✅ PDF 转换能力已具备（Playwright）
- ✅ 生成样式（白底黑字/表格）已较稳定
- ⚠️ `auto-digest.mjs` 仍是“提示词模板流”，不是“最终成稿流”
- ⚠️ output 中历史上混入过非完整版文件，需要持续治理

---

## 2. 格式生成质量评估

### 2.1 Markdown（MD）

**现状**
- `*-complete.md` 文件内容完整，可作为“权威源”。
- 非 complete 版本（如 `hn-daily-YYYY-MM-DD.md`）仍可能含占位符/提示词章节。

**风险**
- 下游（PDF/Pages）若误读非 complete 文件，会把占位符渲染出去。

**建议**
- 统一约定：
  - 最终产物只认 `*-complete.md`
  - 非 complete 文件仅作为中间态，默认清理

---

### 2.2 PDF

**现状**
- 样式已修复为白底黑字。
- 表格渲染已支持（含 `thead/th/td` 样式）。
- BOM 导致标题未渲染问题已修复（转换前去 BOM）。

**风险**
- 如果输入 MD 本身不完整，PDF 会“忠实输出错误内容”。

**建议**
- PDF 前必须跑完整性检查；不通过则中断构建。

---

### 2.3 HTML

**现状**
- 可作为调试产物（debug.html）。
- 目前缺乏统一对外入口，阅读分散。

**建议**
- 对外只通过 GitHub Pages 页面访问（本次已搭建）。

---

## 3. 已发现的历史问题（沉淀）

1. 绝对路径发文件被 sandbox 拒绝（应使用相对路径）
2. Token 出现在文档中触发 GitHub Push Protection
3. BOM 影响 Markdown 标题识别
4. 表格解析缺失导致 PDF 表格异常
5. 生成链路产出“模板/占位符文档”而非最终稿

---

## 4. 本次落地改造

### 4.1 增加 GitHub Pages 展示站点

新增文件：
- `docs/index.html`
- `docs/styles.css`
- `docs/app.js`
- `docs/data/index.json`（构建产物）
- `docs/data/*.md` / `docs/data/*.pdf`（构建产物）

效果：
- 可直接按日期查看每天总结
- 支持在线阅读 Markdown 渲染结果
- 支持一键下载当日 PDF

### 4.2 增加页面数据构建脚本

新增：
- `scripts/build-pages.mjs`

职责：
- 扫描 `output/*-complete.md`
- 生成 `docs/data/index.json`
- 同步对应 PDF 到 `docs/data/`

### 4.3 保持完整性检查机制

已使用：
- `scripts/check-completeness.mjs`

建议在发布流强制执行：
1. 生成 complete.md
2. check-completeness
3. 通过后再转 PDF + 发布 Pages

---

## 5. 发布 GitHub Pages 建议（最小可用）

在 GitHub 仓库设置：
1. Settings → Pages
2. Source 选择 `Deploy from a branch`
3. Branch 选择 `main`，Folder 选择 `/docs`

站点地址通常为：
`https://levix.github.io/hn-daily-skill/`

---

## 6. 后续改进优先级

### P0（立即）
- 将“最终文档标准”固定为 `*-complete.md`
- 在自动流程里禁止发布非 complete 文件

### P1（本周）
- 自动把当天草稿升级成 complete（由 AI 直接填充，不落占位符）
- 在 CI 中加入完整性检查（不通过则 fail）

### P2（后续）
- 在 Pages 增加搜索/标签过滤
- 增加“问题日报”（抓取失败、内容不全、生成耗时）

---

## 7. 验收标准（建议）

每日发布前必须满足：
- [ ] 文章数量 >= 9
- [ ] 无占位符关键词（AI 分析提示词、见文末、TODO、待补充）
- [ ] 每篇包含：中文标题/一句话总结/详细摘要/关键要点/技术洞察
- [ ] PDF 正常渲染（标题/表格/代码块）
- [ ] Pages 可访问且可切换日期

---

结论：
> 仓库已具备稳定输出能力，但要彻底避免“模板文档”回流，必须把 complete 文档作为唯一发布源，并把完整性检查前置为硬门禁。
