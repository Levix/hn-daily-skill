## 1. Do your own writing

- **原文链接**: https://alexhwoods.com/dont-let-ai-write-for-you/
- **HN 评论**: https://news.ycombinator.com/item?id=47573519

### 中文标题
《自己动笔：别让 AI 替你写作》

### 一句话总结
文章主张写作的核心价值在“逼迫思考与建立信任”，若让 LLM 代写，短期省力但会损失理解深度、能力成长和个人可信度。

### 详细摘要
作者认为写文档本质是在回答关键问题：PRD 回答“做什么”，技术方案回答“怎么做”，更高层还要回答“我们到底要达成什么”。这个过程的意义不在“产出一份文本”，而在把模糊问题梳理成结构化认知。作者警告，越来越多 LLM 代写文档会让团队错过这种思考训练。就像健身一样，真正提升能力的是在边界上反复用力；让模型代写相当于“请别人替你锻炼”。此外，作者强调社会层面的代价：当文档明显带有 AI 味道时，读者会怀疑作者是否真正与问题搏斗过，从而削弱领导项目的可信度。文章并不否认 LLM 价值，而是建议把它放在研究、校对、记录转写和发散灵感等环节，让人类保留“核心写作与判断”职责。

### 关键要点
- 写作不是机械产出，而是通过回答问题来澄清认知与目标。
- 让 LLM 代写会削弱“通过写作获得理解”的第一性价值。
- 写作训练像健身，困难与不适本身就是能力增长来源。
- AI 味过重的文档会损害作者在团队中的可信度与领导力印象。
- 更合理的用法是：LLM 辅助研究、校对、转写、发散，而非替代核心思考。

### 技术洞察
文章给出的实操边界很清晰：LLM 适合“高召回、低精度容忍”的任务（如头脑风暴、信息整理），不适合“高责任归属、需体现主判断”的任务（如立项论证、方案取舍、战略写作）。把这两类任务分开，才能同时获得效率和质量。

### 为什么它火了
因为它回应了当前团队的真实焦虑：AI 提效很明显，但“思考外包”也在发生。文章不是反 AI，而是提出“人机分工边界”这件最难却最关键的管理问题，容易引发共鸣。

## 4. Axios compromised on NPM – Malicious versions drop remote access trojan

- **原文链接**: https://www.stepsecurity.io/blog/axios-compromised-on-npm-malicious-versions-drop-remote-access-trojan
- **HN 评论**: https://news.ycombinator.com/item?id=47582220

### 中文标题
《Axios 在 npm 供应链投毒：恶意版本植入跨平台 RAT 投放器》

### 一句话总结
StepSecurity 披露 axios 的两个恶意版本（1.14.1、0.30.4）通过伪装依赖触发 postinstall 木马投放，安装即可能导致系统被远程控制。

### 详细摘要
这起事件是一次针对头部 npm 包的高复杂度供应链攻击。攻击者疑似劫持维护者账号后发布了 `axios@1.14.1` 和 `axios@0.30.4`，并注入从未在源码中使用的依赖 `plain-crypto-js@4.2.1`。该依赖的唯一目的不是提供功能，而是在安装阶段执行 `postinstall` 脚本，下载并投放针对 macOS/Windows/Linux 的二阶段载荷，形成跨平台 RAT 链路。更隐蔽的是，恶意逻辑会“自清理”并伪装成正常包，降低事后取证可见性。事件窗口虽短（数小时内下架），但 axios 体量巨大、传播面广，风险极高。官方与安全社区已进行下架、security hold 与应急通报，建议受影响用户按“已沦陷”标准处置。

### 关键要点
- 受污染版本明确为 `axios@1.14.1` 与 `axios@0.30.4`。
- 攻击路径是“干净主包 + 恶意依赖 + postinstall 执行”，规避源码审查。
- 恶意依赖 `plain-crypto-js@4.2.1` 具备跨平台 RAT 投放能力并回连 C2。
- 恶意链路有预先铺垫（先发干净版本建立历史），体现高组织化与反检测意识。
- 事件已触发 npm 下架与占位替换，但在暴露窗口安装过即应视为高危。

### 技术洞察
这次攻击再次证明“只审主包源码”已不足以防御 npm 供应链风险。真正高风险点在安装生命周期钩子（postinstall）、传递依赖与发布账号安全。企业防线应前移到：依赖版本冻结、安装期脚本审计/禁用、CI 出站流量基线监控、SBOM 与行为检测联动。

### 为什么它火了
因为 axios 是超高下载量基础依赖，影响面横跨前后端与 CI/CD；且攻击手法精细（分支双投毒、短窗口、高隐蔽自清理），把“生态核心包也可瞬时沦陷”的风险直观暴露出来。
