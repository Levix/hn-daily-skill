## 6. A dot a day keeps the clutter away

- **原文链接**: https://scottlawsonbc.com/post/dot-system
- **HN 评论**: https://news.ycombinator.com/item?id=47593556

### 中文标题
《每天一个点，杂乱少一点：低成本实验室库存追踪法》

### 一句话总结
作者用“每开箱一天贴一个圆点”的极简方法，把多年元件使用频率可视化，低成本解决了个人实验室“越囤越乱”的管理难题。

### 详细摘要
文章分享了一套持续运行四年的库存管理实践：统一使用透明标准箱收纳电子元件，并在每次开箱当天给对应箱子贴一个彩色圆点（同一天多次开箱只记一次）。不同颜色对应不同年份，箱体表面的点阵便形成长期“使用热度图”，可快速看出哪些类别高频使用、哪些长期闲置。作者认为这比 RFID、条码、电子表格更适合个人规模：成本仅数美元、无软件依赖、执行摩擦极低、且可扩展。该系统的关键不在“精确到每次取件”，而在用最小记录成本获得可决策的趋势信息，帮助在有限空间下做淘汰与补货平衡，最终让收藏走向稳态而非失控增长。

### 关键要点
- 先解决可见性：淘汰不透明容器，统一透明箱，避免“看不见就忘记”。
- 核心规则极简：每箱每天一个点，记录“使用天数”而非“开启次数”。
- 用颜色映射年份，轻量实现多年度时间序列追踪。
- 点阵直观揭示高频与低频库存，支持清理、保留与采购决策。
- 系统成功关键是低摩擦与可坚持，作者已连续执行四年。

### 技术洞察
这是典型的“行为数据最小化采集”设计：放弃过度精细，换取长期稳定执行。对个人系统而言，可持续性往往比高精度更重要，趋势数据已足够支持高质量决策。

### 为什么它火了
因为它把“库存管理”从复杂工具问题变成人人可复制的习惯动作，成本低、见效快、视觉反馈强，尤其击中创客/工程师“物料堆积却难管理”的普遍痛点。

## 9. EmDash – a spiritual successor to WordPress that solves plugin security

### 中文标题
《EmDash：Cloudflare 打造的“WordPress 精神续作”，重构插件安全模型》

### 一句话总结
EmDash 以 TypeScript + Serverless + 插件沙箱隔离为核心，试图在延续 WordPress 易用生态理念的同时，解决其长期高风险的插件安全问题。

### 详细摘要
Cloudflare 在文章中发布 EmDash（预览版），将其定位为 WordPress 的“精神续作”而非分叉：从零实现、不复用 WordPress 代码、MIT 开源。项目主张把 CMS 带入现代部署范式——默认 Serverless，同时可运行在 Node.js 和自有硬件环境。最关键创新在插件机制：不同于 WordPress 插件常见的高权限直连数据库/文件系统，EmDash 让每个插件在独立隔离环境中运行，并通过 manifest 显式声明能力（capabilities）后才可获得绑定权限，网络访问也可按域名细粒度授权。Cloudflare 认为这能把“安装插件=完全信任”的风险，转变为“安装前可审计、运行时最小授权”的可控模式。文章同时强调其希望兼容 WordPress 的发布体验与生态方向，但从架构层面进行安全与性能重置。

### 关键要点
- EmDash 是从零构建的新 CMS，定位为 WordPress 的精神续作。
- 技术栈以 TypeScript、Astro、Serverless 为核心。
- 插件采用“独立沙箱 + 能力声明 + 最小权限”模型。
- 可在 Cloudflare 或任意 Node.js 环境部署，兼顾托管与自建。
- 目标是在保留易用发布体验的同时，系统性降低插件供应链与越权风险。

### 技术洞察

### 为什么它火了
