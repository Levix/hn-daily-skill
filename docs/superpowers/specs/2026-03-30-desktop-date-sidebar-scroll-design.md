# 桌面端日期栏独立滚动设计

## 背景

当前日报页面采用左右双栏布局，左侧为日期列表，右侧为日报内容。现状下页面的主要滚动体验仍然接近整页滚动，左侧日期栏不会稳定固定在视口中，也没有与右侧内容区完全分离的滚动上下文。

本次改动目标是让桌面端的左侧整栏固定在视口内，并拥有独立的纵向滚动条；右侧内容区也拥有独立的纵向滚动能力。移动端现有顶部横向日期卡片布局保持不变。

## 范围

- 仅调整桌面端布局与滚动行为
- 保持现有 DOM 结构不变
- 保持日期列表数据加载、切换逻辑不变
- 不修改移动端断点下的横向日期卡片交互

## 方案

继续使用当前 `main` 的双栏 grid 结构，不引入 `position: fixed` 或 `position: sticky` 方案。

在桌面端：

- 将 `main` 的高度限制为 `100vh - header 高度`
- 禁止 `main` 自身扩展为随内容无限增高的容器
- 为左侧 `aside` 启用独立的 `overflow-y: auto`
- 为右侧 `section` 启用独立的 `overflow-y: auto`

这样页面在桌面端会形成两个并列的滚动容器：

- 左侧日期栏固定在头部下方的可视区域内，列表过长时仅左栏内部滚动
- 右侧内容区在自身区域内滚动，长文不会推动左栏离开视口

## 布局细节

- `header` 继续保留顶部 sticky 行为
- `main` 的可视高度按照头部实际高度扣减，避免头部与双栏区域叠压
- 左侧栏保持现有宽度与分割线样式
- 右侧内容区保留当前内容排版、代码块、表格等样式

## 风险与兼容性

- 头部高度如果未来调整，`main` 的高度计算需要继续与头部保持一致
- 双滚动容器会改变鼠标滚轮的滚动目标，但这正是本次改动所需行为
- 移动端媒体查询需要显式保留现有单栏布局，避免引入桌面端高度约束

## 验收标准

- 桌面端打开页面时，左侧日期栏始终停留在头部下方视口内
- 日期列表条目超出左栏高度时，左栏出现独立纵向滚动条
- 右侧正文超出可视高度时，仅右侧内容区滚动
- 在桌面端滚动右侧内容时，左侧日期栏位置不随之移动
- 移动端布局和横向日期列表行为保持现状
