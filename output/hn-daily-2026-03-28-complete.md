## 7. Spanish legislation as a Git repo

- **原文链接**: https://github.com/EnriqueLop/legalize-es
- **HN 评论**: https://news.ycombinator.com/item?id=47553798

### 中文标题
《把西班牙法律做成 Git 仓库：每部法律一个 Markdown，每次修法一个 Commit》

### 一句话总结
该项目将西班牙国家层面的整合法律文本结构化为 Git 仓库，用版本控制方式追踪自 1960 年以来的法律修订历史。

### 详细摘要
“Legalize — España” 项目把 BOE（西班牙国家官方公报）开放 API 的整合法律数据转成可工程化处理的代码仓库形态：每部法律对应一个 Markdown 文件，法律修订对应一次独立提交，提交时间与官方发布日期对齐，提交信息包含修订标识与官方来源链接。仓库已收录 8,600+ 部国家级法律文本，覆盖宪法、组织法、普通法、皇家法令法、皇家立法法令等类别。每个文件包含 YAML frontmatter 元数据（标题、标识符、发布日期、最新更新、状态、来源等），便于检索、比对与自动化处理。项目强调自身不创造法律内容，而是在公有领域文本基础上提供结构化、版本历史和元数据层。后续还计划提供 Legalize API，支持程序化查询、过滤、版本比较与变更通知。

### 关键要点
- 以 Git 思维管理法律：法律=文件，修法=commit，天然支持追溯与差异对比。
- 数据来源于 BOE 整合法律 API，覆盖 8,600+ 西班牙国家级法律文本。
- 每次修订独立入库并保留官方时间与来源，增强可审计性与可验证性。
- 文件采用 Markdown + YAML 元数据结构，适合开发者检索、脚本处理和自动分析。
- 项目定位是“结构化与版本化基础设施”，不是法律内容再创作。

### 技术洞察
这是“Law as Code”在公共法规领域的典型实现：用版本控制系统替代静态法条阅读方式，把法规变化转成可 diff、可回溯、可自动化订阅的对象，为合规科技与法律信息工程提供底层数据范式。

### 为什么它火了

这篇文章以逆向分析视角拆解白宫新发布的移动应用。作者称该 App 采用 React Native + Expo（SDK 54）+ Hermes 架构，原生层较薄，业务逻辑主要在 Hermes 字节码中；内容接口来自 whitehouse.gov 的 WordPress 自定义 REST 命名空间，涵盖首页、新闻、直播、议题、图库、社媒等模块。最具争议的发现有两点：其一，内嵌浏览器（WebView）据称会在页面加载时注入 JS/CSS，持续移除 cookie/GDPR/登录墙/订阅墙等元素；其二，尽管配置中存在“去位置权限”相关插件名，APK 内仍包含 OneSignal 位置采集相关代码路径（含前后台间隔、权限门槛、数据上报流程），作者认为该能力并非默认无条件开启，但在满足开关与权限条件后可激活。整体结论是：该应用不仅是信息入口，还包含对外部页面展示与用户数据能力的较强控制面。

### 关键要点
- 逆向结果称 App 技术栈为 React Native + Expo + Hermes，后端是 WordPress 自定义 API。
- 功能定位偏“政务/宣传内容门户”，包含新闻、直播、政策、社媒聚合等。
- 作者称 WebView 注入脚本会主动隐藏第三方网站的同意弹窗、登录墙和付费墙元素。
- APK 内被发现 OneSignal 位置采集基础设施，存在可触发的数据采集与同步链路。
- “去位置”配置与“仍有位置代码”之间的差异，成为文章核心质疑点。

### 技术洞察
这篇分析揭示了现代 App 的一个常见风险面：即使前端宣称最小化权限，真实能力边界仍取决于编译产物与运行时开关。对高敏感应用而言，需同时审计“声明层（配置/商店描述）”与“实现层（二进制/SDK 行为）”的一致性。

### 为什么它火了
因为议题叠加了高政治关注对象、逆向技术细节、隐私与合规争议（同意弹窗/位置追踪），且结论具有强冲击力，容易在技术圈与公共舆论场快速传播。

### 详细摘要
这篇内容是作者对自身抗癌路径的公开说明。其病情为上胸椎 T5 部位骨肉瘤（osteosarcoma）。作者表示，当常规治疗选项用尽且暂无可入组试验后，他开始以“主动掌舵”的方式推进治疗：一方面做更高强度、更多维度的诊断；另一方面探索新治疗思路，并尝试并行推进多种治疗路径，而非单线等待。同时，他将经历系统化输出为文章、演讲材料与公开数据资源（含治疗时间线与大规模可读数据存储），并倡导医疗体系更“患者优先”，减少流程与官僚障碍对个体机会的限制。整体叙事从个人求生延展到方法论与公共协作，强调患者在复杂疾病中的主动权与信息透明价值。

### 关键要点
- 作者确诊为 T5 椎体骨肉瘤，并已走到常规治疗与试验资源不足阶段。
- 在传统路径受限后，转向“最大化诊断 + 新方案探索 + 并行治疗”策略。

- 作者倡导医疗体系更“patient-first（患者优先）”，降低制度性阻碍。
- 提供了公开数据入口与联系方式，意在推动跨患者、跨机构协同。

### 技术洞察
这件事体现了“患者驱动型精准医疗”的雏形：把连续诊断、疗法试探、时间线管理与数据共享整合为一个可迭代系统。它挑战了传统线性医疗流程，强调在高不确定疾病中用并行实验与开放数据提高决策速度和学习效率。
