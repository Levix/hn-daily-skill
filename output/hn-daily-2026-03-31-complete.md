## 1. Do your own writing

- **原文链接**: https://alexhwoods.com/dont-let-ai-write-for-you/
- **HN 评论**: https://news.ycombinator.com/item?id=47573519

### 中文标题
《别让 AI 替你写：写作本质是思考与建立信任》

### 一句话总结
作者认为写作的价值在于澄清问题与锻炼判断力，若把核心写作外包给 LLM，会削弱个人思考深度与团队对你的信任。

### 详细摘要
文章把写作定义为“提出问题并作答”的认知过程：PRD 回答“该做什么”，技术方案回答“怎么做”，更深层则是“我们到底要解决什么”。作者担忧越来越多文档由 LLM 直接代写，这会让人错过最关键的收益——在不确定中形成结构化理解。写作像训练肌肉，真正成长发生在费力、边界不适的过程里；让模型代写，相当于让别人替你锻炼。作者还指出社会层面的副作用：当文档明显是 AI 口吻，读者可能质疑作者是否真正思考过问题，从而削弱其领导力与可信度。文末并未否认 LLM 价值，而是给出边界：可用于调研、校对、转写和发散想法；但核心论证、判断与表达应由人亲自完成。结论是：AI 提效应与“更高思辨密度”同时发生，而不是替代思考。

### 关键要点
- 写作首先是认知工作，不是文字产出流水线。
- LLM 代写会牺牲“通过写作澄清问题”的核心训练价值。
- 写作困难感本身是能力增长来源，类似力量训练。
- AI 味过重的文档会损害作者在协作中的可信度与领导形象。
- 最佳分工是“AI 辅助、人工主导”：让 LLM做发散与校验，人负责结论与责任。

### 技术洞察
文章隐含了一个实用框架：把任务分成“可并行探索型”（检索、脑暴、校对）与“需责任闭环型”（决策文档、方案取舍）。前者可高度自动化，后者必须保留人类主笔以确保可追责与可辩护。

### 为什么它火了
它击中了当下团队普遍痛点——AI 文档产量上升，但思考质量与信任感未必同步提升。该文不是反 AI，而是强调“效率与判断力一起升级”，因此更容易引发广泛共鸣。

## 4. Axios compromised on NPM – Malicious versions drop remote access trojan

- **原文链接**: https://www.stepsecurity.io/blog/axios-compromised-on-npm-malicious-versions-drop-remote-access-trojan
- **HN 评论**: https://news.ycombinator.com/item?id=47582220

### 中文标题
《Axios npm 供应链投毒事件：恶意版本通过隐藏依赖投放跨平台远控木马》

### 一句话总结
StepSecurity 报告称 axios 的 1.14.1 与 0.30.4 被投毒，攻击者利用隐藏依赖的 postinstall 脚本在安装阶段下发跨平台 RAT，已安装者应按“主机已失陷”处置。

### 详细摘要
该事件是一次高复杂度 npm 供应链攻击。攻击者疑似劫持维护者发布权限后，向两个 axios 分支版本（1.x 与 0.x）注入未在源码中使用的依赖 `plain-crypto-js@4.2.1`。该依赖并非功能依赖，而是执行 `postinstall` 的投放器：安装后迅速回连 C2，按系统分发 macOS/Windows/Linux 二阶段载荷。更隐蔽的是，恶意包会自删除并用“干净”元数据伪装，弱化事后排查可见性。事件窗口虽短（数小时内下架），但 axios 体量巨大、传播面广，风险级别极高。报告还披露攻击有预置行为（先发布无害版本建立历史再切恶意版本），体现明显反检测意识。npm 后续已下架受污染版本并对恶意依赖执行安全接管。

### 关键要点
- 受影响版本：`axios@1.14.1`、`axios@0.30.4`。
- 攻击核心：隐藏依赖 `plain-crypto-js@4.2.1` + `postinstall` 自动执行。
- 恶意行为：连接 C2、下发跨平台二阶段载荷、执行后自清理伪装。
- 手法特征：双分支近时投毒、提前铺垫“干净包历史”、短窗口高覆盖。
- 应急建议：若安装过受影响版本，按“已被入侵”处理并全面轮换凭据。

### 技术洞察
这起事件再次证明“主包源码无恶意”不等于安全。真正高危面在传递依赖与安装生命周期钩子。有效防线应包括：锁定依赖版本、默认禁用/审计 postinstall、CI 出站流量异常检测、最小权限执行环境与 SBOM 持续校验。

### 为什么它火了
因为 axios 是超高下载量基础依赖，任何短时间投毒都可能触发大规模连锁风险；且此次攻击在执行链路、隐蔽性与节奏控制上都非常专业，给整个 JS 生态敲响了警钟。
